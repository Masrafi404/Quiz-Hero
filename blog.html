1: What is the difference between map and filter and foreach and find?

Answer: map and filter are higher-order functions used to manipulate collections in functional programming. map applies a function to each element in a collection and returns a new collection of the same size with the transformed values, while filter returns a new collection containing only the elements that satisfy a given predicate. forEach is a method used to iterate over the elements of a collection and perform a specified action on each element, but it does not return anything. find is also a method used to iterate over the elements of a collection, but it returns the first element that satisfies a given predicate, or null if no element satisfies the predicate.


2: What is the difference between local storage and session storage and cookies?

Answer: Local storage and session storage are both mechanisms for storing data on the client-side in a user's browser, but they differ in terms of their lifespan and scope. Local storage persists even after the browser is closed and is available to all pages in the same origin, while session storage is only available for the duration of the current session and is limited to the current window or tab. Cookies are also used to store data on the client-side, but they have additional features like expiration dates and can be used for tracking and authentication purposes.


3: What is Event lopp and how does is work?

Answer: An event loop is a programming pattern that enables the handling of asynchronous events in a non-blocking way. It works by continuously checking for events in a queue and executing any associated callbacks or event handlers. This allows the program to perform other tasks while waiting for events to occur. Event loops are commonly used in languages like JavaScript, Python, and Node.js, for web applications and other event-driven systems.
